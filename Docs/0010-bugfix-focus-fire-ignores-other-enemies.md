# 0010. 버그 수정: 지정 적 공격 중 다른 적을 먼저 공격하는 문제

## 날짜
2026-07-07

## 요청
지금 적유닛 선택공격을 했을때 앞에 다른 적유닛이 있으면 무시하고 가는게 아니라 그냥 땅 공격이랑 같은 방식으로 작동하는거 같네 다른 적 유닛은 무시하고 선택된 그 유닛을 공격시키고 싶어

## 원인
`AttackRange.GetPreferredTarget()`가 지정된 추격 대상(`orderedTarget`)이 아직 사거리 트리거 안에 들어오지 않았을 때 `GetClosestEnemy()`로 폴백해서, 지정한 적이 아닌 다른 아무 적이나 사거리에 들어오면 그걸 공격해버렸음.

## 답변 / 변경사항
1. **`AttackRange.cs`**: 지정 대상이 있으면 그 대상이 트리거 안에 있을 때만 반환하고, 없으면(트리거 밖) 다른 적으로 대체하지 않고 `null` 반환 — 지정 대상 외 어떤 적도 무시.
2. **`UnitController.cs`**: 추격 중 "사거리 안인지" 판단을 `attackRange.HasEnemyInRange`(아무 적이나 감지되면 true) 대신, 지정한 대상과의 실제 거리로 직접 판단하도록 변경 — 무관한 다른 적이 트리거에 있어도 추격 이동이 멈추지 않도록 함.

## 변경 파일

- `Assets/Scripts/Unit/UnitController.cs`
