# 유닛/건물 수치 정리

> 최종 갱신: 2026-07-22
> `UnitDataSO`(`New Unit Data SO.asset`)와 `BuildingDataSO`(`New Building Data SO.asset`)에 적힌 값을 그대로 정리한 문서입니다. 유닛은 `doc/0205`~`doc/0210` 작업 이후로 SO 값이 실제 게임 값과 동일합니다(프리팹 인스펙터도 SO와 동기화되어 있음). `공격 방식`/`고유 보너스`/`공격 범위(공중 이동 여부)`만 SO가 아니라 유닛 프리팹의 `UnitController` 값을 그대로 옮겨온 것입니다(SO에는 해당 필드가 없음).

## 데미지 시스템 (참고)

**장갑 타입** — 유닛의 고유 추가 데미지가 노리는 대상을 구분:
- 경장갑: 보병, 경차량
- 중장갑: 전차, 대형유닛

**크기 타입** — 공격 방식별 데미지 배율을 결정:
- 소형: 작은 보병, 경차량
- 중형: 중형 유닛(일반형, 배율 보정 없음)
- 대형: 대형 유닛(공격 방식에 따라 배율 보정 큼)

**공격 방식별 크기 배율** (`DamageMultiplierTable.asset`):

| 공격 방식 | 소형 | 중형 | 대형 |
|---|---|---|---|
| 소총 | 100% | 80% | 60% |
| 폭발 | 70% | 100% | 130% |
| 레이저 | 100% | 100% | 100% |
| 화염 | 130% | 100% | 60% |

최종 데미지 = `공격력(연구 보너스 포함) × 크기 배율 × (대상이 고유보너스 대상 장갑타입이면 1+보너스%) → 반올림 → 대상 방어력 감산 → 최소 1 보장`

## 참고/미구현 사항

- **공격 범위(지상/공중 타겟팅 제약)는 설계 의도일 뿐 코드에 구현되어 있지 않습니다.** 현재 `AttackRange`는 `"Enemy"` 태그만 보고 지상/공중 구분 없이 아무 유닛이나 공격 가능합니다. 위 표의 "공격 범위"는 실제 제약이 아니라 기획 의도를 적어둔 것입니다.
- **SkyLancer의 `isAirUnit`이 아직 0(지상)**이라, "공중유닛"이라는 설계와 달리 실제로는 지상 유닛처럼 내비메시 위를 이동합니다. 프리팹에서 직접 고쳐야 합니다.
- **Sharpshooter의 "연구소 필요" 생산 조건은 미구현.** 현재 생산 가능 여부는 `UnitData.tier`(건물 종류)만으로 결정되고, 연구소 완공 여부 같은 추가 조건(테크 요구사항)을 검사하는 로직 자체가 없습니다.
- **IFV 레인저의 "수송(유닛 탑승)" 기능 미구현.** 코드 전체에 탑승/하차 관련 로직이 없습니다.
- **지원기(Support Ship)는 완전 구상 단계.** 프리팹/SO 데이터 없음, 아군 버프 시스템 자체가 프로젝트에 없음.
- 더 상세한 "설계 값 vs 과거 실측값" 비교 이력은 `Docs/UnitBalanceReference.md` 참고(이 문서 작성 시점 기준으로는 그 문서에 있던 불일치들이 대부분 해소된 상태입니다).
