# 0032. 유닛 타입별 Avoidance Priority 추천

- 날짜: 2026-07-08

## 요청 내용

> 내가 적용시킬게 근데 어떤유닛들이 우선순위가 높으면 좋을꺼 같아?

([[0031-ramp-unit-jitter-investigation]]에서 제안한 formation 오프셋 수정을 사용자가 직접 적용하기로 하고, 보조책으로 언급했던 `navMeshAgent.avoidancePriority` 값을 유닛 타입별로 어떻게 나눌지에 대한 후속 질문)

## 조사 내용 / 답변

Unity `NavMeshAgent.avoidancePriority`는 0~99 범위이며 **숫자가 낮을수록 우선순위가 높다**(잘 안 비켜줌, 남들이 비켜줌). 숫자가 높을수록 우선순위가 낮다(잘 비켜줌).

### 타입별 권장 값 (우선순위 높은 순)

1. **대형/저속 차량** (Pulsar Tank, Ranger Infantry Fighting Vehicle, Goliath급) — 20~30. 덩치가 커서 우회가 어렵고 한번 밀리면 재정렬에 오래 걸리므로 남들이 비켜주는 편이 전체 흐름에 유리.
2. **일반 보병** (Assault Trooper 등) — 50 (기본값). 몸집이 작고 민첩해 우회가 쉬움.
3. **일꾼(Worker Drone)** — 60~80. 수가 많고 전투 가치가 없어 아무 때나 비켜줘도 손해가 적음.

### 좁은 경사로 진동 문제와의 관계

동일 타입 유닛끼리(예: 트루퍼끼리)만 경사로에 몰리는 경우엔 타입 계층이 전부 동률이라 위 값만으로는 도움이 안 됨. 이 경우엔 **스폰 시 개체별로 작은 랜덤 편차(±5~10)를 얹는 것**이 핵심 처방 — 동률이 깨져야 "누가 비킬지"가 프레임마다 흔들리지 않고 한 줄로 정렬되며 진동이 줄어든다.

결론: **타입 기반 기본값(차량 < 보병 < 일꾼 순으로 숫자 증가) + 개체별 랜덤 지터**를 함께 적용하는 것을 추천. [[0031-ramp-unit-jitter-investigation]]의 formation 오프셋 수정이 먼저 들어가면 애초에 같은 지점에 안 몰리므로, 이 우선순위 튜닝은 보조 효과 정도로 체감될 것.
