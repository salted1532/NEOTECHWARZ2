# 0029 - Rules 폴더 생성 및 매 요청마다 규칙 확인/적용

**날짜:** 2026-07-08

## 요청 내용

현재 적용된 규칙(메모리에 저장된 규칙들) 내용을 프로젝트 내 `Rules` 폴더를 만들어 정리해두고, 앞으로 사용자가 하는 매 요청마다 그 규칙을 확인한 뒤 답변을 작성하고 행동할 것.

## 조사 내용

현재 이 프로젝트에 적용 중인 규칙은 Claude Code의 영구 메모리(`~/.claude/projects/.../memory/`)에 저장된 2개:

1. **docs-session-logging-rule** — 매 세션의 모든 질문/명령과 답변/변경 내역을 `doc/NNNN-*.md`로 기록 (번호는 세션 넘어 이어짐, `Docs/`와 `doc/`는 별개)
2. **confirm-before-implementing-rule** — 코드/에셋 변경 요청은 먼저 `doc/NNNN-*.md` 제안서 작성 → 사용자 확인 → 승인 후에만 실제 파일 변경. 순수 Q&A는 게이트 미적용.

이 두 규칙은 메모리 시스템에 원본이 있고, 프로젝트 자체에는 사람이 읽을 수 있는 사본이 없었음. 요청은 이를 프로젝트 안(`Rules/`)에도 두어, 프로젝트만 열어봐도 규칙을 확인할 수 있게 하고, 매 요청 시작 시 이 폴더를 참고하도록 하는 것.

## 계획된 변경 (기존 코드 → 변경 코드)

### 기존 코드
`Rules/` 폴더 없음 (프로젝트 루트에 규칙 사본이 존재하지 않음)

### 변경 코드

**신규 파일: `Rules/README.md`**
```markdown
# Rules

이 폴더는 Claude Code가 이 프로젝트에서 작업할 때 매 요청마다 확인하고 따르는 규칙 목록입니다.
원본은 Claude Code의 영구 메모리(`~/.claude/projects/.../memory/`)에 있으며, 이 폴더는 프로젝트 내에서
바로 확인할 수 있도록 만든 사본/요약본입니다.

- [session-logging-rule.md](session-logging-rule.md) — 세션 로깅 규칙
- [confirm-before-implementing-rule.md](confirm-before-implementing-rule.md) — 구현 전 확인 규칙
```

**신규 파일: `Rules/session-logging-rule.md`**
```markdown
# 세션 로깅 규칙

모든 사용자 질문/명령과 그 결과(답변/변경 내역)는 `doc/NNNN-<영문-slug>.md`
(4자리 번호, 시간순, 소문자 doc/ 폴더)에 기록한다.

- 파일 구성: 날짜, 요청 내용, 조사 내용(필요시), 코드 변경(기존 코드 → 변경 코드 블록,
  코드가 실제로 바뀐 경우만), 요약/남은 작업, 변경된 파일
- 번호는 세션을 넘어 계속 이어짐 — 새로 쓰기 전 `doc/[0-9][0-9][0-9][0-9]-*.md`로 최신 번호 확인
- `Docs/`(대문자, 스크립트별 레퍼런스 문서)와 `doc/`(소문자, 세션 로그)는 별개 — 혼동 금지
- 순수 Q&A/조사 요청은 코드 diff 없이 서술 부분만 작성
```

**신규 파일: `Rules/confirm-before-implementing-rule.md`**
```markdown
# 구현 전 확인 규칙

프로젝트 코드/에셋을 변경하는 요청이 들어오면 즉시 구현하지 않는다.

1. 먼저 `doc/NNNN-*.md` 제안서를 작성한다 (요청 내용, 조사 내용,
   계획된 기존 코드 → 변경 코드 블록 포함)
2. 사용자에게 진행 여부를 명시적으로 물어본다
3. 승인되면 실제 프로젝트 파일(`Assets/...`, 프리팹, 스크립트 등)에 적용한다.
   초안이 곧 기록이 되며, 실제 구현이 제안과 달라진 경우에만 문서를 갱신한다
4. 거절/변경되면 초안 문서를 그에 맞게 수정하거나 폐기한다

- 순수 Q&A/조사 요청(코드 변경 없음)은 이 확인 절차 대상 아님
- 이미 승인된 변경 내의 사소한 후속 수정(오타 등)은 재확인 불필요
```

## 앞으로의 동작 방식 (제안)

매 요청 시작 시:
1. `Rules/` 폴더(또는 메모리)의 규칙을 확인
2. 요청이 코드/에셋 변경을 수반하는지 판단
   - 수반 시 → `doc/NNNN-*.md` 제안서 먼저 작성 → 사용자 확인 후 구현
   - 순수 Q&A/조사 → 바로 답변, 이번 요청도 세션 로그(`doc/`)에는 기록
3. 규칙 자체가 바뀌면 `Rules/` 폴더와 메모리 양쪽을 함께 갱신

## 요약/영향받는 파일

- 신규 생성 예정: `Rules/README.md`, `Rules/session-logging-rule.md`, `Rules/confirm-before-implementing-rule.md`
- 기존 메모리 파일(`docs_session_logging.md`, `confirm_before_implementing.md`)은 원본으로 유지, 필요시 "프로젝트 내 Rules/ 폴더에도 사본 존재"를 언급하도록 갱신 예정
- 아직 실제 파일 생성 전 — 사용자 확인 대기 중
