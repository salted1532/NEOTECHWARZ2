# 0139. 유니티 정상 실행 확인 및 깃허브 저장

## 날짜
2026-07-16

## 요청/보고
유니티가 정상적으로 켜졌고, 이전까지의 진행 과정을 만들고 깃허브에 저장했다는 보고.

## 참고
- `git` CLI가 현재 세션의 PowerShell PATH에 없어(`git : 이 명령어를 찾을 수 없습니다`) 직접 커밋 로그로 확인은 못 했다 — GitHub Desktop 등 별도 도구로 저장한 것으로 추정.
- `doc/0138`에서 되돌렸던 `GetComponentInChildren` 수정은, `doc/0139`(이 문서) 시점에 다시 확인해보니 실제 원인이 아니었다(원인은 `Library/Bee`의 유령 락, `doc/0138` 이후 대화에서 확인). 즉 지금 코드(`CaptureSystem.cs`)는 여전히 `GetComponent<TerritoryZone>()`(같은 오브젝트만 검색)로 남아 있어서, `TerritoryZone`이 자식 오브젝트(`Capture_territory`)에 있는 현재 프리팹 구조에서는 점령 시 외곽선 색이 여전히 안 바뀌는 상태 — 재적용 여부는 사용자 확인 필요.

## 변경 파일
없음 (상태 확인/보고만).

## 다음 단계
`GetComponentInChildren` 수정을 다시 적용할지 사용자에게 물어봐야 함 — 이번 컴파일 멈춤의 진짜 원인이 그 코드가 아니었다는 게 확인됐으므로, 다시 넣어도 안전할 가능성이 높음.
