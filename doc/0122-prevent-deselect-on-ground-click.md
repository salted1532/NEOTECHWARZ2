# 0122. 빈 땅 좌클릭 시 선택 해제 방지

**날짜:** 2026-07-15

## 요청 내용

유닛이나 건물을 선택한 후 아무 땅이나 좌클릭했을 때 `DeselectAll`이 호출되어 선택이 풀리는 것을 막아달라는 요청. 선택된 상태를 그대로 유지하고 싶어함.

## 조사 내용

`Assets\Scripts\UserControl\UserControl.cs`의 `HandleLeftClick()` 메서드는 클릭 대상을 우선순위대로 검사한다:

1. 유닛 클릭 → 선택/강제공격 후 `return`
2. 적 클릭 → 선택/공격 후 `return`
3. 건물 클릭 → 선택/강제공격 후 `return`
4. 땅 클릭 → **명령 대기 상태(Move/Attack/Patrol/Rally/BuildingMove)일 때만** 명령을 실행하고 `return`. 대기 상태가 `None`(평상시 좌클릭)이면 아무 것도 하지 않고 그냥 통과함.
5. 광물/가스 클릭 → 선택 후 `return`
6. 위 어느 것도 아니면(즉, 평상시 좌클릭으로 땅을 클릭한 경우가 여기로 흘러들어옴) Shift가 안 눌려있으면 `DeselectAll()` 호출

즉, 지금 구조상 "빈 땅을 좌클릭"하면 4번 블록에서 아무 처리도 안 하고 통과한 뒤 6번 블록에서 걸려 선택 해제가 발생한다. 이 6번 조건에 `!clickedGround`를 추가하면, 땅을 클릭했을 때는 선택 해제를 하지 않고 선택 상태를 그대로 유지하게 된다. (드래그 박스로 새로 선택하거나, 유닛/적/건물/자원을 직접 클릭하는 경우는 기존 로직이 그대로 처리하므로 영향 없음.)

## 계획된 코드 변경

**파일:** `Assets\Scripts\UserControl\UserControl.cs` (라인 387~391)

### 기존 코드
```csharp
        // 6. 아무것도 아닌 곳 클릭 = 선택 해제
        // (Shift를 누른 채 빈 바닥에서 드래그를 시작한 경우엔, 곧이어 시작될 드래그 선택이
        //  기존 선택에 "추가"되어야 하므로 여기서 기존 선택을 지우지 않는다)
        if (!Input.GetKey(KeyCode.LeftShift))
            rtsUnitController.DeselectAll();
```

### 변경 코드
```csharp
        // 6. 아무것도 아닌 곳 클릭 = 선택 해제
        // (Shift를 누른 채 빈 바닥에서 드래그를 시작한 경우엔, 곧이어 시작될 드래그 선택이
        //  기존 선택에 "추가"되어야 하므로 여기서 기존 선택을 지우지 않는다)
        // (땅 클릭은 선택 해제하지 않는다 - 평상시 좌클릭으로 빈 땅을 눌러도 기존 선택을 유지)
        if (!Input.GetKey(KeyCode.LeftShift) && !clickedGround)
            rtsUnitController.DeselectAll();
```

## 요약/영향받는 파일

- 이 변경 후에는 좌클릭으로는 땅(레이어: `layerGround`)을 클릭해도 더 이상 선택이 해제되지 않는다. 선택 해제는 이제 다른 방식(예: 드래그로 빈 공간 선택, 새 유닛/건물 클릭 등 다른 대상 클릭)으로만 일어난다.
- 실제로 아무 레이어(유닛/적/건물/자원/땅)도 맞지 않는 클릭(예: 맵 바깥 하늘)에서는 여전히 선택이 해제된다.
- 영향받는 파일: `Assets\Scripts\UserControl\UserControl.cs`
