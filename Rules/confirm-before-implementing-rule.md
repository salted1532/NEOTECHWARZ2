# 구현 전 확인 규칙

프로젝트 코드/에셋을 변경하는 요청이 들어오면 즉시 구현하지 않는다.

1. 먼저 `doc/NNNN-*.md` 제안서를 작성한다 (요청 내용, 조사 내용,
   계획된 기존 코드 → 변경 코드 블록 포함)
2. 사용자에게 진행 여부를 명시적으로 물어본다
3. 승인되면 실제 프로젝트 파일(`Assets/...`, 프리팹, 스크립트 등)에 적용한다.
   초안이 곧 기록이 되며, 실제 구현이 제안과 달라진 경우에만 문서를 갱신한다
4. 거절/변경되면 초안 문서를 그에 맞게 수정하거나 폐기한다

- 순수 Q&A/조사 요청(코드 변경 없음)은 이 확인 절차 대상 아님
- 이미 승인된 변경 내의 사소한 후속 수정(오타 등)은 재확인 불필요
